# Review Harness — "Harm Implies Action Deviation"

Independent reproduction and diagnostic probes for the empirical claims of the
paper *Harm Implies Action Deviation: A Formal Systematization of Behavioral
Compromise Detection in LLM Agent Ecosystems* (submitted to IEEE TDSC).

This harness accompanies a referee review. Its purpose is to let anyone
reproduce, from the artifact the authors shipped, the two facts that most
affect how the headline in-policy result should be read:

1. the `0.151 -> 0.968` "provenance lever" is, on the reconstructed AgentDojo
   data, the difference between ignoring and reading a ground-truth taint label
   the data generator wrote into every attack record; and
2. the three "competing defenses" compared against the method reduce, on this
   dataset, to the same boolean predicate.

Neither point says the data are fake — the trajectories are built from
AgentDojo's own published ground-truth annotations. They are statements about
how the reconstruction and its labels bound what the numbers can mean.

## Contents

```
review_harness/
  data/runs.jsonl              ground-truth reconstruction (726 records:
                               97 benign + 629 attacks), copied from the
                               authors' package for self-contained probes
  probes/
    probe_circularity.py       Probe 1: phi5's 0.968 == reading back the label
    probe_baseline_identity.py Probe 2: the 3 baselines are one predicate
    probe_reproduce.py         Probe 3: reproduce 0.968 + exact CI, and point
                               at the authors' code for the full 0.151/0.968
  run_reproduction.sh          runs all probes; optionally invokes authors' code
  requirements.txt             stdlib only; scipy optional (for CIs)
```

## Quick start

```bash
python3 probes/probe_circularity.py       --runs data/runs.jsonl
python3 probes/probe_baseline_identity.py --runs data/runs.jsonl
python3 probes/probe_reproduce.py         --runs data/runs.jsonl
# or all at once:
bash run_reproduction.sh
```

Optional — regenerate the full `0.151 / 0.968` pair with the authors' own
calibrated ensemble (requires their package):

```bash
ABD_REPO=/path/to/agent-behavioral-detection bash run_reproduction.sh
```

## What each probe establishes (verified output)

**Probe 1 — circularity.** Of 629 attacks, 609 contain at least one
`from_untrusted=True` call to a privileged sink; `phi5` fires on exactly those
609 -> detection `0.9682`, matching the paper's `0.968`. It fires on `0` of 97
benign trajectories. The number is recovered with no detector and no learning,
by counting the generator's own labels.

**Probe 2 — provenance, not the rule, carries detection.** Three mechanisms
are implemented from distinct decision rules, each using side information derived
*independently of the attack labels* (AgentArmor's privileged-sink set is defined
semantically from tool names, not from `from_untrusted`). Measured on 629
attacks: Task Shield proxy `0.161`, AgentArmor proxy `0.935`, f-secure IFC proxy
`0.968`. The two provenance-aware rules track each other closely (agree on 608
of 629, differing only where the sole untrusted call hits a non-side-effecting
tool), while the provenance-blind task-alignment rule sits at ~0.16. What
separates the high detectors from the low one is access to provenance, not the
specific signature — the paper's own conclusion, made precise. Because the sink
set is not built from the labels, this agreement is measured, not forced by
construction.

**Probe 3 — reproduction.** Confirms N=629 attacks / 97 benign, that every
attack is `success=True` by construction, and reproduces the with-provenance
`0.968` (k=609, exact Clopper-Pearson 95% CI `[0.951, 0.980]`). The
no-provenance `0.151` is regenerated by the authors' calibrated ensemble via
`run_reproduction.sh` (a hand-rolled exact-match proxy over-fires and is
deliberately not used to fake that number).

## Scope and honesty notes

- The probes read `data/runs.jsonl`, which is the authors' shipped
  reconstruction. They do not re-run any LLM; neither did the reconstruction.
- `probe_reproduce.py` intentionally does **not** hand-derive the `0.151`
  figure, because a naive provenance-blind proxy does not reproduce it; the
  faithful path is the authors' calibrated ensemble.
- These diagnostics support the *interpretation* of the results (how to read
  the provenance lever and the baseline comparison), not an accusation that the
  data are invalid.
